My CSS Cheatsheet - stuff you keep forgetting
Topic What to remember
Pseudo-classes You can use pseudo classes to add interactivity to any selector. It's always written as colon, followed by a name ex:
p:hover { background-color: lime; }.
Now whenever the mouse hovers over a paragraph element, that paragraph will have a lime colored background.
Opacity: In CSS, Opacity has a value from 0 to 1.
0 is transparent, 1 is opaque, and in between is a scale between the two.
Border-radius You can change an element's box to a circle if you use border:radius.
The easiest way to do this is to set the height and width equal to each other, then set border-radius: 50%.
Padding: You can think of padding as a way to increase the inside space of an element.
Also important! When you use padding, unless you specify, it will add the amount that you code in to each side of the box. So, coding in padding: 50px; means that in total, it will add 100px to the entire box (up, left, down, right)
Padding, just like padding, can be increased on each side separately or all together, via 1, 2, 3, or 4 values, depending on whether you want some sides to be the same.
Margin: You can think of Margin as a way to increase the surrounging area, or outside space of an element.
Margin, just like padding, can be increased on each side separately or all together, via 1, 2, 3, or 4 values, depending on whether you want some sides to be the same.
You can use margin: to center an element with margin: 0 auto; but remember if you do this the element must have a width.
Vertical vs Horizontal dimensions Remember! With the box model, horizontal dimensions add, but vertical dimensions do not!
Vertical dimensions simply take the value of the highest px (or other unit of measure)
Overflow: This property controls what happens to content that spills or overflows outside of its box. It can either be hidden, scrolled, or visible(default)
If hidden, any content that overflows will be hidden from view
If set to scroll, a scrollbar will be added to the element's box so that the rest of the content can be viewed by scrolling.
Visbility: 3 values, hidden, visible, and collapse. If you choose to use hidden, remember that the web page will only hide the elements. There will still be an empty space for where the element is supposed to be.
Note, if you use display:none, it will do the same thing as visibility:hidden except the empty space for the element will also disappear.
Box-sizing: border-box; It's good practice to use box-sizing: border-box; to avoid dimensional issues with your web page. This is because in the default model (content-box) box dimensions are affected by border thickness and padding. Border-box isn't.
Position: You can use position to change the positioning of elements. Position has 5 possible values. static(default), relative, absolute, fixed, and sticky.
relative (stay within normal flow of document. when user scrolls elements scroll too.) allows you to position an element relative to its default static position. In tandem with this, you specify from the top, bottom, left or right you want the element to move, on separate css declarations.
absolute (removed from document flow. stay at specified offset position.) allows you to position an element relative to its closest positioned parent element. Then you can offset it top, left, bottom or right just like relative. Note that if position is set to absolute, all other elements on the page will ignore the element and act like it isn't on the page.
fixed (removed from document flow. stay at specified offset position.) allows you to position an element in a fixed place. It will stay on the page and will follow the user as they scroll the document. Then you can offset it top, left, bottom, or right just like relative or absolute. This technique is often used with navigation bars.
sticky allows you to position an element keeping an element in the document flow as user scrolls, but it will then stick to a specified position as page is scrolled further.
static (stay within normal flow of document. when user scrolls elements scroll too.) Default value.
Display: In-lineheight and width cannot be specified. does not require a new line after each element. Think em, strong, a.
Block height and width can be specified. a new line is created after each element. Think h1-h6, p, div, footer.
Inline-block height and width can be specified. however a new line is not created after each element. Think img.
Float Used to simply move an element all the way to the left or right. Commonly used for wrapping text around an image. (not an ideal declaration). Floated elements must have a width.
Clear Specifies how elements behave when they bump into each other on the page. left, right, both, none. ex. clear: left, left side of the element will not touch any other element within the same containing element.
